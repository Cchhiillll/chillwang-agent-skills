# Article Templates

## 1. Project introduction article
Use when explaining what a project is and why it matters.

Suggested structure:
1. What it is
2. What problem it solves
3. Core capabilities
4. Main advantages
5. Trade-offs / boundaries
6. Best-fit scenarios
7. My judgment

## 2. Methodology / workflow article
Use when extracting reusable process knowledge.

Suggested structure:
1. The background problem
2. The current way of working
3. End-to-end workflow
4. Why it works
5. Main lessons
6. Next optimization step

## 3. Product analysis article
Use when deriving product insights from another project.

Suggested structure:
1. Define the object clearly
2. Identify what layer it serves
3. Explain why it matters
4. Extract 3-5 key lessons
5. Show how to apply the lessons locally
